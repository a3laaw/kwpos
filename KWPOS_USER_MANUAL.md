## 4. نقاط البيع (POS)

### الوضع العادي (Standard Mode)
- شريط جانبي كامل + قوائم منسدلة
- بحث بالاسم أو الباركود
- بطاقات منتجات بصور
- سلة كاملة مع خصم + ضريبة + توصيل
- محدّد طريقة الدفع (نقدي/بطاقة/تحويل)
- ربط عميل بالهاتف
- طباعة إيصال حراري

### الوضع السريع (Express Mode)
- **شاشة كاملة** بدون قوائم
- **مدخل باركود كبير** (auto-focus دائم)
- **بطاقات منتجات كبيرة** (4/3/2 أعمدة)
- **سلة مبسطة**: صورة + اسم + كمية + سعر + حذف
- **أزرار دفع كبيرة**: "نقدي" و "بطاقة"
- **خيارات أكثر** (قابل للطي): هاتف العميل، خصم، توصيل
- **قارئ باركود عالمي** (USB HID)
- **زر فتح درج النقدية**

### التبديل بين الوضعين
- زر "الوضع السريع / Standard Mode" في شريط الأدوات
- SALES و CASHIER يبدؤون في الوضع السريع
- ADMIN يبدأ في الوضع العادي
- التفضيل يُحفظ في قاعدة البيانات

### إجراءات البيع
| الإجراء | الطريقة |
|---------|---------|
| إضافة منتج للسلة | نقر البطاقة أو مسح الباركود |
| تغيير الكمية | أزرار + / - في السلة |
| حذف صنف | زر سلة المهملات |
| تفريغ السلة | Esc (في الوضع السريع) |
| إتمام البيع | F2 أو Ctrl+Enter أو زر "دفع" |
| طباعة الإيصال | تلقائي (إذا مفعّل) أو زر "طباعة" |

### ضريبة المنتج
- كل منتج له **نسبة ضريبة خاصة** (في صفحة المنتج)
- الضريبة تُحسب **لكل صنف** على حدة
- المنتجات بنسبة 0% لا تُضاف لها ضريبة

## 5. الفواتير

### قائمة الفواتير
- عرض كل الفواتير (الأحدث أولاً)
- بحث برقم الفاتورة
- فلترة بالحالة (مدفوعة/مرتجعة)
- **طباعة إيصال حراري** (80mm)
- **طباعة فاتورة A4** (مع شعار الشركة)
- **استرداد/مرتجع** (Refund)
- **تبديل** (Exchange)

### أنواع الطباعة
| النوع | الحجم | الاستخدام |
|------|-------|-----------|
| إيصال حراري | 80mm | كاشير سريع |
| فاتورة A4 | A4 | فاتورة رسمية للعميل |
| إيصال تبديل | 80mm | عمليات التبديل |

## 6. المخزون

### إدارة المنتجات
- **إضافة منتج**: اسم، باركود، تصنيف، صورة، أسعار، نسبة ضريبة
- **تعديل منتج**: كل الحقول قابلة للتعديل
- **حذف منتج**: يتطلب تأكيد
- **بحث**: بالاسم أو الباركود (فوري)
- **فلترة**: بالتصنيف، المخزون المنخفض
- **استيراد/تصدير Excel**
- **طباعة باركود**: ملصقات للمنتجات

### حقول المنتج
| الحقل | الوصف |
|-------|-------|
| الاسم | اسم المنتج (مطلوب) |
| الباركود | 13 رقم حسب التصنيف |
| التصنيف | قسم المنتج |
| الصورة | JPG/PNG/WEBP (تُصغّر تلقائيًا) |
| سعر التكلفة | سعر الشراء |
| سعر البيع | سعر التجزئة |
| سعر الجملة | سعر الجملة (0 = نفس التجزئة) |
| سعر الشركات | سعر الشركات (0 = نفس التجزئة) |
| نسبة الضريبة | % (افتراضي 0) |
| حد إعادة الطلب | الكمية التي تنبّه النقص |
| الكمية المثلى | كمية الطلب المثلى |

### المخازن
- **مخازن متعددة** مع كميات مستقلة
- `Product.quantity` = مجموع كل المخازن (تلقائي)
- **تحويلات بين المخازن** (شحنة صادرة + واردة)
- **تقرير حركة المخزون** (8 أنواع حركة)

### أنواع حركة المخزون
| النوع | الوصف | تأثير الكمية |
|-------|------|-------------|
| SALE | بيع | - |
| REFUND | مرتجع مبيعات | + |
| EXCHANGE | تبديل | ± |
| PURCHASE_INVOICE | فاتورة مشتريات | + |
| PURCHASE_RETURN | مرتجع مشتريات | - |
| TRANSFER_OUT | تحويل صادر | - |
| TRANSFER_IN | تحويل وارد | + |
| STOCK_TAKE | جرد | ± |
| SPOT_CHECK | جرد أعمى | ± |

### الجرد
- **جرد دوري**: جرد كامل مع تعديل الكميات
- **جرد أعمى (Spot Check)**: جرد مفاجئ بدون عرض الرصيد
- يتطلب موافقة المدير للاعتماد

## 7. المشتريات

### أوامر الشراء (Purchase Orders)
- **إنشاء أمر شراء**: اختيار المورد + المنتجات + الكميات
- **الموافقة**: MANAGER/ADMIN يوافق على الأمر
- **الاستلام**: تحديث المخزون + تحديث أسعار التكلفة
- **الإلغاء**: إلغاء الأمر
- **إنشاء تلقائي**: اقتراح أوامر شراء للأصناف منخفضة المخزون

### فواتير المشتريات
- **فاتورة مشتريات**: من المورد
- **تكلفة الوصول (Landed Cost)**: جمارك + شحن + رسوم أخرى
- **ترحيل الفاتورة**: إنشاء قيد محاسبي + تحديث المخزون
- **مرتجع مشتريات**: إرجاع بضائع للمورد

### مدفوعات الموردين
- **سداد مورد**: دفع للمورد + قيد محاسبي
- **كشف حساب المورد**: كل المعاملات + الرصيد
- **تصدير PDF/Excel**

## 8. الموردين

### إدارة الموردين
- **إضافة مورد**: اسم، هاتف، بريد، عنوان
- **تعديل/حذف مورد**
- **كشف حساب المورد**: كل الفواتير + المدفوعات + الرصيد
- **ربط منتج بمورد**: مورد افتراضي لكل منتج

## 9. العملاء

### إدارة العملاء
- **إضافة عميل**: اسم، هاتف، عنوان، نوع
- **أنواع العملاء**: تجزئة (RETAIL)، جملة (WHOLESALE)، شركات (CORPORATE)
- **كشف حساب العميل**: كل المشتريات + المدفوعات + الرصيد
- **نقاط الولاء**: نقاط تلقائية + طبقات

### نظام نقاط الولاء
| الطبقة | النقاط المطلوبة |
|--------|----------------|
| BRONZE | 1,000+ |
| SILVER | 5,000+ |
| GOLD | 10,000+ |

- **منح النقاط**: 1 نقطة لكل 1 وحدة عملة (تلقائي عند البيع)
- **استرداد النقاط**: كخصم قبل الدفع
- **تقرير الولاء**: أعلى العملاء بالنقاط

## 10. المحاسبة

### شجرة الحسابات
- **17 حساب جاهز** مع أكواد محاسبية
- **أكواد الحسابات**:
  - 1010: النقدية (Cash)
  - 1020: البنك (Bank)
  - 2010: ذمم دائنة (Accounts Payable)
  - 4010: إيراد مبيعات (Sales Revenue)
  - 5070: تكلفة البضاعة المباعة (COGS)

### القيود المحاسبية
- **قيد مزدوج** (Double Entry) تلقائي
- عند كل عملية بيع/شراء/مرتجع
- داخل المعاملة (atomic) — إذا فشل القيد، تتراجع العملية

### ميزان المراجعة
- عرض أرصدة كل الحسابات
- فلترة بالنوع (أصول، خصوم، إيرادات، مصروفات)

### دفتر الأستاذ العام
- حركة كل حساب بالتفصيل
- رصيد افتتاحي + حركة + رصيد ختامي
- تصدير PDF/Excel

### القوائم المالية
| القائمة | الوصف |
|---------|------|
| الميزانية العمومية | أصول + خصوم + حقوق ملكية |
| قائمة التدفقات النقدية | دخلة + خرج + صافي |
| قائمة الدخل | إيرادات - مصروفات = صافي الربح |
| كشف حساب العميل | كل معاملات العميل |
| كشف حساب المورد | كل معاملات المورد |
| تقرير VAT | ضريبة المخرجات + المدخلات |

### المصروفات
- **رواتب**: اسم الموظف + المبلغ + التاريخ
- **مصروفات إدارية**: إيجار، اشتراكات، تسويق، أخرى
- كل مصروف يُنشئ قيدًا محاسبيًا تلقائيًا

## 11. التقارير والتحليلات

### تقرير المبيعات العام
- فلاتر: تاريخ، منتج، تصنيف، طريقة دفع، مصدر
- KPIs: إجمالي المبيعات، التكلفة، الربح، هامش الربح
- رسوم بيانية: مبيعات يومية، طرق الدفع، الأقسام
- جدول تفصيلي بالمنتجات
- طباعة (جداول فقط) + تصدير PDF/Excel

### تحليلات المبيعات (5 أنواع)
| التحليل | الوصف |
|---------|------|
| نظرة عامة | ملخص شامل |
| الأكثر مبيعًا | Top products بالكمية |
| الأصناف الراكدة | منتجات لم تُبع |
| التكاليف | أعلى المنتجات تكلفة |
| الربحية | أعلى المنتجات هامش ربح |

### مصفوفة الأداء
- أداء كل قسم + كل منتج
- الربحية + معدل الدوران + أيام الركود
- فلترة بالتاريخ + التصنيف

### طباعة التقارير
- زر **طباعة** يفتح نافذة نظيفة
- يطبع **الجداول فقط** (بدون قوائم/فلاتر/أزرار)
- عنوان التقرير + التاريخ في الترويسة

## 12. الورديات

### فتح وردية
- الكاشير يفتح وردية مع **رصيد افتتاحي**
- تسجيل كل عملية بيع خلال الوردية

### إغلاق وردية
- **المبالغ المتوقعة**: نقدي + K-Net + Visa (من المبيعات)
- **المبالغ الفعلية**: يدخلها المشرف (الجرد الفعلي)
- **الفرق (Variance)** = الفعلي - المتوقع
- يتطلب موافقة MANAGER/ADMIN

### موافقة المدير
- مراجعة الفروقات
- اعتماد أو رفض
- إنشاء قيد محاسبي للفرق

## 13. التبديل والاستبدال

### عملية التبديل
- **البحث عن الفاتورة الأصلية** برقمها
- **تحديد الأصناف المرتجعة** (بكميات سالبة)
- **تحديد الأصناف الجديدة** (بكميات موجبة)
- **التسوية**: 
  - إذا الجديد > القديم → العميل يدفع الفرق
  - إذا القديم > الجديد → المتجر يسترد الفرق
- **طريقة الدفع**: نقدي/بطاقة/تحويل
- **طباعة إيصال تبديل** (80mm)
- حد أقصى 14 يوم من تاريخ الفاتورة الأصلية

## 14. التسعير والعروض

### إدارة الأسعار
- تعديل أسعار البيع لكل المنتجات
- 3 طبقات: تجزئة، جملة، شركات
- تحذير عند السعر أقل من التكلفة
- سجل تغييرات الأسعار (Audit Log)

### العروض والخصومات
- **عرض بنسبة مئوية**: خصم % على منتج/تصنيف
- **عرض بمبلغ ثابت**: خصم بمبلغ
- **تاريخ بداية ونهاية** للعرض
- تفعيل/إيقاف العرض

## 15. الباقات والتركيبات

### الباقات (Bundles)
- **إنشاء باقة**: اختيار منتجات + كميات + سعر باقة
- **حساب تلقائي**: التكلفة، الربح، نسبة الخصم
- **موسمية**: تاريخ بداية ونهاية
- مثال: عطر + بخور + مبخرة بسعر مخفّض

### التركيبات (Compositions)
- **إنشاء تركيبة**: اختيار مكونات خام + كميات + منتج ناتج
- **حساب التكلفة**: تكلفة الدفعة + تكلفة الوحدة
- **إنتاج دفعة**: خصم المكونات من المخزون + إضافة الناتج
- مثال: 5جم دهن عود + 3جم بخور = "عود ملكي" 8جم

## 16. الجرد الأعمى (Spot Check)

- **جرد مفاجئ** لمنتج حساس
- **بدون عرض الرصيد الدفتري** ( Blind Count)
- إدخال الكمية الفعلية
- حساب الفرق تلقائيًا
- إنشاء قيد محاسبي للفرق

---

## 17. التدقيق (Audit)

### سجل التدقيق
- **كل العمليات الحساسة** تُسجل تلقائيًا
- **الأفعال المسجلة**:
  - SALE_CREATED (إنشاء بيع)
  - SALE_REFUNDED (مرتجع)
  - SALE_EXCHANGED (تبديل)
  - VOID_ITEM (حذف صنف من السلة)
  - DRAWER_OPEN (فتح درج النقدية)
  - MANAGER_APPROVAL (موافقة مدير)
  - COMPOSITION_PRODUCED (إنتاج تركيبة)
  - BUNDLE_CREATED/UPDATED/DELETED
  - PURCHASE_INVOICE_POSTED

### تحليل الحذف
- **نسبة الحذف لكل كاشير** (Void Rate)
- كشف السلوك المشبوه
- تصنيف: طبيعي / مشبوه

## 19. الإعدادات

### بيانات الشركة
- **اسم الشركة** (يظهر في الفواتير)
- **العنوان** (يظهر في الفواتير)

- **الرقم الضريبي** (يظهر في فاتورة A4)
- **شعار الشركة** (يظهر بجانب الاسم)
- تُحفظ في `localStorage` (لكل جهاز)

## 21. الأمان والصلاحيات

### المصادقة
- **NextAuth.js** مع JWT (7 أيام)
- كلمات المرور مشفرة بـ **bcrypt**
- جلسة تُجدّد تلقائيًا

### حماية API
- كل API route محمي بالدور
- `getCurrentUser()` + `hasRole()` في كل route
- النسخ الاحتياطي يتطلب كلمة مرور المدير

### قفل الصفوف (Row Locking)
- `SELECT FOR UPDATE` على المخزون الحساس
- يمنع البيع المتزامن لنفس الصنف
- يمنع التحويلات المتزامنة

### Audit Log
- سجل كامل لكل العمليات
- **بدون سر افتراضي** (fail-closed)
- يتطلب `AUDIT_INTERNAL_SECRET` في بيئة Vercel
