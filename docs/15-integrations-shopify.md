# 15 — التكاملات: ربط شوبيفاي (Shopify)

> دليل تدريبي مبسّط لشاشة «التكاملات» في نظام KWPOS وللربط مع منصّة شوبيفاي. يشرح ماهية التكامل، خطوات الإعداد، المتغيّرات البيئية المطلوبة، حالة الاتصال، تدفّق استيراد الطلبات، تدفّق مزامنة المنتجات، الأخطاء المحتملة، والقيود الحالية. مكتوب لغير المختصين تقنياً مع الإشارة للملفات المصدرية.

الملفات الرئيسية:

| الملف | دوره |
|---|---|
| `src/components/integrations/integrations-view.tsx` | شاشة التكاملات (بطاقة شوبيفاي + دليل الإعداد) |
| `src/lib/shopify.ts` | عميل Admin API (REST) لشوبيفاي |
| `src/app/api/shopify/status/route.ts` | فحص حالة الاتصال |
| `src/app/api/shopify/sync-products/route.ts` | مزامنة المنتجات (Shopify → المحلي) |
| `src/app/api/shopify/import-orders/route.ts` | استيراد الطلبات (Shopify → فواتير مبيعات) |
| `src/lib/i18n/settings.ts` | نصوص الواجهة (مفاتيح `int*`) |

## 1. ما هو تكامل شوبيفاي؟

شوبيفاي (Shopify) منصّة تجارة إلكترونية شائعة تُستخدم لإنشاء متاجر إلكترونية. تكامل KWPOS مع شوبيفاي يربط متجرك الإلكتروني بنظام النقاط والمخزون المحلي، فيُتيح:

| الميزة | الاتجاه | الوصف |
|---|---|---|
| **مزامنة المنتجات** | Shopify → KWPOS | سحب منتجات المتجر الإلكتروني وإضافتها أو تحديثها في مخزونك المحلي |
| **استيراد الطلبات** | Shopify → KWPOS | سحب آخر 50 طلباً وإنشائها كفواتير مبيعات محلية تلقائياً |
| **عرض حالة الاتصال** | KWPOS | فحص دوري لاتصال الـ API |

> **ملاحظة مهمّة:** التكامل **اختياري** — النظام يعمل بكامل وظائفه بدون شوبيفاي. عند تفعيله فقط تظهر أزرار المزامنة والاستيراد. لم يظهر في الكود أي اتجاه عكسي (KWPOS → Shopify) — لا يوجد دفع للأسعار أو المخزون من النظام إلى شوبيفاي.

## 2. الإعداد — 4 خطوات

عند فتح شاشة التكاملات ولم يُضبط التكامل بعد، يظهر **دليل الإعداد** (`SetupGuide` في `integrations-view.tsx`) يحتوي 4 خطوات مرقّمة:

| # | الخطوة | التفاصيل |
|---|---|---|
| 1 | **افتح إعدادات شوبيفاي** | من لوحة تحكم شوبيفاي: `Apps ← Develop apps`. رابط المساعدة: https://www.shopify.com/admin/apps/development |
| 2 | **أنشئ تطبيقاً مخصصاً** | اضغط `Create an app`، أعطه اسماً، ثم من `Configuration` أضف الصلاحيات: `read_products` و`read_orders` |
| 3 | **ثبّت التطبيق واحصل على المفتاح** | اضغط `Install app`، ثم انسخ **Admin API access token** (يبدأ بـ `shpat_` ويظهر مرة واحدة فقط) |
| 4 | **أضف البيانات إلى ملف `.env`** | املأ المتغيّرين التاليين ثم أعد تشغيل الخادم |

### المتغيّرات البيئية المطلوبة (SHOPIFY_*)

يقرأها `src/lib/shopify.ts` من `process.env`:

```bash
# .env
SHOPIFY_STORE_DOMAIN=your-store.myshopify.com
SHOPIFY_ACCESS_TOKEN=shpat_xxxxxxxxxxxxxxxxxxxxxxxx
```

| المتغيّر | الوصف | مثال |
|---|---|---|
| `SHOPIFY_STORE_DOMAIN` | نطاق متجر شوبيفاي (بدون `https://`) | `my-store.myshopify.com` |
| `SHOPIFY_ACCESS_TOKEN` | رمز وصول Admin API | `shpat_abc123...` |

> ⚠️ **لا تُخزَّن هذه القيم في قاعدة البيانات** — تُقرأ من `.env` فقط. لتعديلها عليك تحرير الملف ثم إعادة تشغيل الخادوم. لا توجد واجهة لتعديلها من داخل النظام.

### نسخة الـ API

يستخدم النظام إصدار Admin API الثابت: `2024-01` (مُعرَّفة كـ `API_VERSION` في `src/lib/shopify.ts`).

### بنية الطلب

كل طلبات شوبيفاي تستخدم:

```
GET https://{SHOPIFY_STORE_DOMAIN}/admin/api/2024-01/{path}
Headers:
  X-Shopify-Access-Token: {SHOPIFY_ACCESS_TOKEN}
  Accept: application/json
```

## 3. حالة الاتصال (Connection Status)

عند تحميل الصفحة، تستدعي الواجهة `GET /api/shopify/status` لعرض الحالة.

| الحقل | الوصف |
|---|---|
| `configured` | `true` إن كان المتغيّران معبَّئين، وإلا `false` |
| `domain` | نطاق المتجر (`SHOPIFY_STORE_DOMAIN`) أو `null` |
| `message` | رسالة عربية: «متصل بمتجر شوبيفاي» أو «لم يتم إعداد التكامل بعد...» |

### عرض الحالة في الواجهة

| الحالة | الشارة | اللون |
|---|---|---|
| مُعدّ (configured = true) | ✅ «متصل» | أخضر |
| غير مُعدّ (configured = false) | ⚠️ «غير مربوط» | كهرماني |

زر التحديث ↻ في الزاوية العلوية يُعيد فحص الحالة.

### الصلاحية

كل مسارات `/api/shopify/*` تتطلب دور `ADMIN` فقط (راجع `src/app/api/shopify/status/route.ts`). باقي الأدوار ترى الصفحة في القائمة الجانبية لكن لا يستطيعون تنفيذ المزامنة.

> **لم يظهر في الكود** أي فحص دور أوسع (مثل السماح لـ `OWNER`). حتى `OWNER` لا يستطيع تشغيل المزامنة، فقط `ADMIN`. هذا قد يُعدَّل مستقبلاً.

## 4. مزامنة المنتجات (Sync Products)

الـ API: `POST /api/shopify/sync-products` (الملف `src/app/api/shopify/sync-products/route.ts`).

### تدفّق العمل

1. الخادوم يتصل بـ `GET /admin/api/2024-01/products.json?limit=100`.
2. يجلب حتى 100 منتج من شوبيفاي (المتغيّر `limit = 100` في `fetchShopifyProducts`).
3. يتأكد من وجود فئة افتراضية باسم `Shopify` في قاعدة البيانات (يُنشئها إن لم توجد).
4. لكل منتج شوبيفاي:
   - يستخرج أول `variant` (النوع/المقاس).
   - يجلب `sku` (كود SKU)، `barcode`، `price`، `inventory_quantity`.
   - **قاعدة المطابقة:** يطابق بالباركود (= SKU) إن وُجد، وإلا بالاسم.
   - إن وُجد المنتج محلياً → يُحدّث (الباركود، السعر، الكمية، الفئة).
   - إن لم يوجد → يُنشئ منتجاً جديداً تحت فئة `Shopify`.

### الحقول المُحدَّثة عند وجود المنتج

| الحقل | المصدر من شوبيفاي |
|---|---|
| `barcode` | `variant.sku` أو `variant.barcode` |
| `salePrice` | `variant.price` (إن > 0) |
| `quantity` | `variant.inventory_quantity` (إن > 0) |
| `categoryId` | الفئة المحلية الحالية أو فئة `Shopify` |

### الحقول الافتراضية عند إنشاء منتج جديد

| الحقل | القيمة |
|---|---|
| `name` | عنوان المنتج في شوبيفاي |
| `barcode` | `variant.barcode` أو `variant.sku` |
| `categoryId` | معرّف فئة `Shopify` |
| `quantity` | `variant.inventory_quantity` أو 0 |
| `reorderLevel` | 5 (افتراضي ثابت) |
| `costPrice` | 0 (لم يظهر في الكود كيفية جلب التكلفة من شوبيفاي) |
| `salePrice` | `variant.price` أو 0 |
| `unit` | «قطعة» (افتراضي ثابت) |

> ⚠️ **تكلفة الشراء = 0** دائماً عند المزامنة. شوبيفاي لا يُرسل تكلفة الشراء في Admin API القياسية، وعلى المستخدم إدخالها يدوياً بعد المزامنة لحساب الربحية بدقة.

### الردّ

```json
{
  "ok": true,
  "fetched": 50,
  "created": 12,
  "updated": 35,
  "errors": []
}
```

| الحقل | المعنى |
|---|---|
| `fetched` | عدد المنتجات المجلوبة من شوبيفاي |
| `created` | عدد المنتجات المُنشأة محلياً |
| `updated` | عدد المنتجات المُحدَّثة محلياً |
| `errors` | قائمة بأخطاء لكل منتج (تُقطَع إلى أول 10) |

## 5. استيراد الطلبات (Import Orders)

الـ API: `POST /api/shopify/import-orders` (الملف `src/app/api/shopify/import-orders/route.ts`).

### تدفّق العمل

1. الخادوم يتصل بـ `GET /admin/api/2024-01/orders.json?limit=50&status=any`.
2. يجلب آخر 50 طلباً من شوبيفاي.
3. لكل طلب شوبيفاي:
   - يبني رقم فاتورة محلياً بصيغة `SHP-{shopify_order_id}` (مثل `SHP-123456789`).
   - يبحث عن فاتورة محلية بهذا الرقم — إن وُجدت يُعدّ الطلب «مستورَداً سابقاً» ويُتجاوز (`skipped`).
   - لكل بند (`line_item`) في الطلب: يطابق المنتج محلياً بالـ SKU/الباركود ثم بالاسم.
   - إن لم يُطابَق المنتج، يُضاف خطأ `«{order_name}: منتج غير موجود — {item_title}»` ويُتجاوز البند.
4. يُنشئ فاتورة مبيعات محلية ببيانات الطلب:

| الحقل المحلي | المصدر من شوبيفاي |
|---|---|
| `invoiceNo` | `SHP-{order.id}` |
| `customerName` | `order.customer.display_name` أو «عميل شوبيفاي» |
| `subtotal` | مجموع `qty × price` لبنود مطابَقة |
| `taxRate` | 0 (لا تستخرج ضريبة شوبيفاي) |
| `taxAmount` | 0 |
| `discount` | 0 |
| `total` | `order.total_price` (الأصلي من شوبيفاي) |
| `paid` | نفس `total` (يُعتبر مدفوعاً) |
| `paymentMethod` | `TRANSFER` (تحويل) افتراضياً |
| `userId` | معرّف الـ ADMIN الذي استدعى الاستيراد |

### ملاحظات مهمّة

- **لا يُخصَّم المخزون** تلقائياً عند الاستيراد. الطلب يُسجَّل كفاتورة لكن لا تُعدَّل كميات المنتجات. هذا تصرف مختلف عن فاتورة POS اليدوية التي تخصم المخزون.
- **لا تُنشأ قيود محاسبية** تلقائياً للطلبات المستوردة. الفاتورة تُسجَّل في جدول المبيعات فقط.
- الطلبات بدون أي بنود مطابَقة تُسجَّل كفاتورة فارغة (subtotal = 0) لكن بقيمة `total` كاملة من شوبيفاي. هذا يُبقي سجلّ الطلب ظاهراً للتجار.
- `paymentMethod = TRANSFER` ثابتة — لم يظهر في الكود أي منطق لاستخراج طريقة الدفع الفعلية من بيانات الطلب.

### الردّ

```json
{
  "ok": true,
  "fetched": 50,
  "imported": 18,
  "skipped": 32,
  "errors": ["#1001: منتج غير موجود — تيشيرت أحمر"]
}
```

| الحقل | المعنى |
|---|---|
| `fetched` | عدد الطلبات المجلوبة من شوبيفاي |
| `imported` | عدد الطلبات المُنشأة كفواتير محلية جديدة |
| `skipped` | عدد الطلبات المُستوردة سابقاً (موجودة برقم `SHP-...`) |
| `errors` | قائمة بأخطاء لكل طلب (تُقطَع إلى أول 10) |

## 6. أزرار الواجهة

عند ضبط التكامل (`configured = true`)، تظهر في البطاقة لوحتان:

| اللوحة | الزر | الدالة |
|---|---|---|
| **مزامنة المنتجات** | «مزامنة الآن» | `handleSyncProducts` → `POST /api/shopify/sync-products` |
| **استيراد الطلبات** | «استيراد الآن» | `handleImportOrders` → `POST /api/shopify/import-orders` |

أثناء العمل يظهر مؤشّر دوران (`Loader2`) ويُعطَّل الزر. بعد النجاح يظهر Toast:

- **مزامنة المنتجات:** `جُلب: N • أُنشئ: N • حُدّث: N`
- **استيراد الطلبات:** `جُلب: N • أُستورد: N • تخطّي: N`

تحت اللوحتين تظهر منطقة «ملاحظات» صفراء عند وجود أخطاء (حتى 5 أخطاء لكل عملية)، تذكِّر المستخدم بالبنود التي لم تُطابَق.

## 7. الأخطاء المحتملة

### أخطاء الـ API

| الكود | الحالة | السبب |
|---|---|---|
| `unauthorized` | 401 | غير مسجَّل الدخول |
| `forbidden` | 403 | ليس دور `ADMIN` |
| `shopify-not-configured` | 400 | المتغيّران البيئيان فارغان |
| `shopify-http-401` | – | رمز الوصول خاطئ أو منتهي |
| `shopify-http-403` | – | التطبيق لا يملك صلاحيات `read_products` أو `read_orders` |
| `shopify-http-404` | – | نطاق المتجر خاطئ |
| `shopify-http-429` | – | تجاوز حدّ المعدّل (Rate Limit) من شوبيفاي |
| `shopify-http-5xx` | – | خطأ في خوادم شوبيفاي نفسها |
| `sync-failed` | 500 | خطأ غير متوقّع أثناء المزامنة |
| `import-failed` | 500 | خطأ غير متوقّع أثناء الاستيراد |

### أخطاء المطابقة (تُجمَع في `errors` ولا تُوقف العملية)

| الرسالة | السبب |
|---|---|
| `{order_name}: منتج غير موجود — {item_title}` | بند الطلب لا يطابق أي منتج محلي |
| `{product_title}: {error}` | فشل إنشاء/تحديث منتج محلي محدَّد |

### نصائح لتفادي الأخطاء

| المشكلة | الحل |
|---|---|
| المنتجات لا تتطابق | تأكد أنّ SKU في شوبيفاي = الباركود في KWPOS (مطابقة بالـ SKU أولاً) |
| الطلبات تتكرَّر | لن تتكرَّر — الـ API يتحقق من `SHP-{id}` قبل الإنشاء |
| تجاوز حدّ المعدّل | لا تستدعِ المزامنة أكثر من مرة كل بضع دقائق |
| فاتورة بـ 0 بنود | المنتجات في الطلب لا تطابق أيّ منتج محلي — حدّث أسماء أو باركودات المنتجات المحلية |

## 8. القيود الحالية (لم تظهر في الكود)

| الميزة | الحالة |
|---|---|
| دفع المنتجات من KWPOS → شوبيفاي (تحديث المخزون على المتجر الإلكتروني) | ❌ لم يظهر في الكود |
| مزامنة الطلبات بشكل دوري تلقائي (Cron job) | ❌ لم يظهر في الكود — المزامنة يدوية فقط |
| استيراد العملاء من شوبيفاي إلى KWPOS | ❌ لم يظهر في الكود |
| استيراد الطلبات بأكثر من 50 | ❌ ثابتة بـ `limit=50` |
| جلب تكلفة الشراء من شوبيفاي | ❌ ثابتة بـ 0 |
| ربط طريقة الدفع (نقدي/بطاقة/تحويل) بالطريقة الفعلية في شوبيفاي | ❌ ثابتة بـ `TRANSFER` |
| ربط طلب شوبيفاي بعميل موجود في KWPOS | ❌ يُنشأ اسم نصّي فقط دون ربط FK |
| Webhooks (تنبيه فوري عند طلب جديد في شوبيفاي) | ❌ لم يظهر في الكود |
| مزامنة المنتجات المتغيّرة (Variants) كمنتجات منفصلة | ❌ يُؤخذ أول variant فقط |

### ما الذي يلزم لإضافة هذه الميزات؟

لو أردت إضافة أيّ من الميزات أعلاه، يحتاج التطوير إلى:

1. **مزامنة عكسية (Push):** كتابة دالة `updateShopifyProduct()` تستدعي `PUT /admin/api/2024-01/products/{id}.json` مع صلاحية `write_products`.
2. **Webhooks:** تسجيل Webhook عبر `POST /admin/api/2024-01/webhooks.json` لنوع `orders/create`، وإنشاء مسار `/api/shopify/webhook` للتحقّق من التوقيع وتنفيذ المنطق.
3. **جلب تكلفة الشراء:** إمّا إدخال يدوي بعد المزامنة، أو استخدام Shopify Plus API (محدود) أو الحقل `inventory_cost` عبر GraphQL Admin API.
4. **Cron job:** استخدام `node-cron` أو Vercel Cron لإطلاق `sync-products` و`import-orders` كل ساعة مثلاً.

## 9. الأمان والخصوصية

- رمز الوصول `SHOPIFY_ACCESS_TOKEN` حسّاس — لا يُكشف أبداً للواجهة (يُستخدم فقط في الخادوم).
- صلاحيات التطبيق محدودة بـ `read_products` و`read_orders` فقط (لا يوجد write) — لا يستطيع النظام تعديل متجر شوبيفاي.
- كل مسارات الـ API تتطلب مصادقة ودور `ADMIN`.

## 10. أسئلة شائعة

**س: هل أستطيع مزامنة المنتجات من شوبيفاي إلى KWPOS دون إنشاء فئة «Shopify»؟**
ج: لا، الكود يُنشئ فئة افتراضية باسم `Shopify` إن لم توجد ويضع كل المنتجات المُ synced فيها. تستطيع بعدها نقل المنتجات لفئة أخرى يدوياً.

**س: ماذا يحدث لو حذفت منتجاً في شوبيفاي بعد مزامنته؟**
ج: الكود لا يحذف المنتجات المحلية عند الحذف من شوبيفاي — يُ同步 (sync) الإضافات والتحديثات فقط. حذف المنتج المحلي عملية يدوية.

**س: هل أستطيع ربط طلب شوبيفاي بعميل موجود مسبقاً في KWPOS؟**
ج: لم يظهر في الكود منطق ربط بمعرّف عميل محلي. اسم العميل يُسجَّل كنص حر في حقل `customerName` فقط.

**س: متى تُستورد الطلبات تلقائياً؟**
ج: أبداً. الاستيراد يدوي عبر زر «استيراد الآن». لا يوجد Webhook أو Cron في الكود الحالي.

**س: هل أستطيع تخصيص رقم الفاتورة المستوردة؟**
ج: لا، الرقم ثابت بصيغة `SHP-{shopify_order_id}` ويُستخدم للتحقق من عدم التكرار.
