## 1. شاشة الدخول (Login Screen)

تظهر شاشة الدخول في `src/components/auth/login-screen.tsx` لأي مستخدم غير مسجّل. وهي منقسمة إلى لوحتين:

| اللوحة | المحتوى |
|---|---|
| **لوحة العلامة (يسار/يمين حسب اللغة)** | شعار النظام، عنوان رئيسي، وصف موجز، ثلاث ميزات بصور أيقونية (سلة، مخزون، صناديق). |
| **لوحة النموذج (الجهة الأخرى)** | بطاقة فيها حقلان: **البريد أو اسم المستخدم**، و**كلمة المرور**، ثم زر «تسجيل الدخول». |

### 1.1 الحقول

| الحقل | النوع | مثال |
|---|---|---|
| البريد / اسم المستخدم | نص (`text`) | `admin` أو `admin@demo.com` |
| كلمة المرور | كلمة مرور (`password`) | `••••••••` |

### 1.2 تسجيل دخول مرن (باسم المستخدم أو البريد)

كما في `src/lib/auth.ts` (الدالة `authorize`)، يقبل النظام ثلاث طرق للإدخال:

1. **البريد الكامل** (مثل `admin@demo.com`) — مطابقة تامة.
2. **اسم المستخدم** (مثل `admin`) — يطابق الجزء قبل `@` من البريد.
3. **الاسم المعروض** (مثل `أحمد`) — مطابقة حساسة لحالة الأحرف لحقل `name`.

> **للموظفين الميدانيين:** يكفي كتابة `cashier` أو `sales` ثم كلمة المرور — لا حاجة لتذكّر البريد الكامل.

### 1.3 إنشاء مدير تلقائي عند أول تشغيل (Auto-Bootstrap Admin)

كما في `src/app/page.tsx`:

```ts
// Auto-seed: if no users exist AND BOOTSTRAP_ADMIN_PASSWORD env is set,
// create a default admin with that password. If the env var is NOT set,
// no admin is created (fail-safe — the user must seed via /api/seed).
```

| الشرط | النتيجة |
|---|---|
| قاعدة البيانات فارغة **و** متغيّر البيئة `BOOTSTRAP_ADMIN_PASSWORD` مُعيَّن | يُنشأ مستخدم `admin@demo.com` بدور `ADMIN` وكلمة المرور المُحدَّدة. |
| قاعدة البيانات فارغة **و** المتغيّر غير مُعيَّن | لا يُنشأ مستخدم (fail-safe — يجب تشغيل `/api/seed` يدوياً). |
| يوجد مستخدمون أصلاً | لا يحدث شيء — تُعرض شاشة الدخول. |

> **أمان:** لا يوجد كلمة مرور افتراضية ضعيفة. النظام يفشل بأمان بدل إنشاء حساب ضعيف.

### 1.4 بعد النجاح

- تُستدعى `signIn("credentials", {...})` من `next-auth/react`.
- يُخزَّن JWT صالح لمدة **7 أيام** (راجع `src/lib/auth.ts`).
- يُعاد تحميل الصفحة (`router.refresh()`) لتُعرض `AppShell` بدلاً من شاشة الدخول.

### 1.5 حسابات تجريبية

تظهر في `src/components/auth/login-screen.tsx` أزرار سريعة للتعبئة المسبقة (Quick Fill):

| الدور | البريد التجريبي |
|---|---|
| مدير النظام | `admin@demo.com` |
| مدير | `manager@demo.com` |
| محاسب | `accountant@demo.com` |
| موظف مبيعات | `sales@demo.com` |
| أمين مخزن | `warehouse@demo.com` |
| كاشير | `cashier@demo.com` |

> **ملاحظة:** زر التعبئة يملأ اسم المستخدم فقط (الجزء قبل `@`) لتذكير المستخدم بأن تسجيل الدخول بـ «اسم المستخدم» مدعوم.

## 8. التبديل بين اللغتين (RTL / LTR)

مصدر: `src/components/i18n-context.tsx`.

### 8.1 زر التبديل

أيقونة 🌐 في الشريط العلوي (`LangToggle`). نقرة واحدة تبدّل اللغة فوراً.

### 8.2 ما الذي يحدث عند التبديل؟

| العنصر | السلوك |
|---|---|
| اللغة المحفوظة | تُكتب في `localStorage["erp-locale"]` كـ `ar` أو `en`. |
| اتجاه الصفحة | يتغيّر `<html dir="rtl">` إلى `<html dir="ltr">` أو العكس (في `useEffect`). |
| لغة الصفحة | يتغيّر `<html lang="ar">` إلى `en` أو العكس. |
| اتجاه الشريط الجانبي | ينزلق Sheet الجوال من اليمين في العربية، من اليسار في الإنجليزية. |
| اتجاه الأسهم | في القائمة المجمَّعة، السهم المغلقة يشير نحو المحتوى (`ChevronLeft` في RTL، `ChevronRight` في LTR). |
| التذييل | يتبدّل اتجاهه تلقائياً. |

### 8.3 القيمة الافتراضية

- الخادم (SSR) يعرض دائماً بالعربية (مطابقة لـ `layout.tsx`) لتفادي وميض الاتجاه.
- العميل: إذا كان لدى المستخدم لغة محفوظة (مثل `en`)، يُطبَّق فوراً عند التحميل دون وميض.

### 8.4 تنسيق التواريخ حسب اللغة

كما في `src/components/currency-context.tsx`:

| لغة الواجهة | تنسيق التواريخ |
|---|---|
| `ar` | `<country-locale>-u-nu-latn` (أشهر عربية، أرقام لاتينية) |
| `en` | `en-GB` (أشهر إنجليزية، أرقام لاتينية) |

> **ملاحظة:** العملة تبقى مرتبطة بالدولة الفعّالة، لا بلغة الواجهة.

## 9. عرض العملة والدولة

مصدر: `src/components/currency-context.tsx` و `src/lib/settings.ts` و `src/lib/countries.ts`.

### 9.1 الدولة الفعّالة

- تُحفظ في جدول `Setting` بمفتاح `active_country` (افتراضي `KW`).
- تُقرأ من الخادم في `getActiveCountryCode()` مع تخزين مؤقت 5 ثوانٍ.
- تُمرَّر إلى `AppShell` كـ `country: CountryConfig`، ثم تتوفّر لكل المكوّنات عبر `useCountry()`.

### 9.2 العرض في الشريط العلوي

على الحاسوب (`sm+`):

```
[🇰🇼 د.ك]   ←  في خلفية رمادية خفيفة، مع تلميحة باسم الدولة
```

### 9.3 تنسيق الأرقام (`useFmt()`)

| الدالة | الاستخدام |
|---|---|
| `fmt.currency(123.45)` | تنسيق مبلغ بالعملة المحلية (`د.ك 123.450` للكويت، `ر.س 123.45` للسعودية...). |
| `fmt.number(1234.5)` | تنسيق رقم عادي. |
| `fmt.date("2024-01-15")` | تاريخ قصير حسب لغة الواجهة. |
| `fmt.dateTime(...)` | تاريخ ووقت. |
| `fmt.symbol` | رمز العملة (`د.ك`). |
| `fmt.taxRate` | نسبة الضريبة الافتراضية للدولة (مثل `15` للسعودية). |
| `fmt.taxLabel` | تسمية الضريبة («ضريبة القيمة المضافة» أو «بدون ضريبة»). |

### 9.4 الخانات العشرية

تختلف حسب الدولة (من `currencyDecimals`):

- 3 خانات: الكويت، البحرين، عُمان، الأردن، تونس.
- 2 خانة: السعودية، الإمارات، قطر، مصر، المغرب، الجزائر.
- 0 خانة: العراق.

## 10. التنقّل بين الوحدات

### 10.1 الطرق الأربع للتنقّل

### 10.2 حالة العرض (View State)

- تُحفظ الشاشة الحالية في Zustand store (`useAppStore`) ومُستمَّدة في `localStorage`.
- عند إعادة فتح التطبيق، يُسترجع آخر شاشة عرضتها — **إلا** إذا لم تعد مسموحة لدورك، فتُعاد توجيهك إلى الشاشة الافتراضية (راجع `app-shell.tsx`).

### 10.3 حماية الصلاحيات على الواجهة

حتى لو حاول المستخدم فتح شاشة محجوبة يدوياً:

1. العنصر لا يظهر أصلاً في القائمة الجانبية.
2. `app-shell.tsx` يفحص `allowedViews.includes(view)` قبل عرض المكوّن.
3. إذا لم تكن مسموحة، يُعرض `null` وتُصحَّح الحالة في `useEffect`.
