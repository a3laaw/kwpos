# 11 — الوحدة المحاسبية (Accounting)

> دليل تدريبي مبسّط للوحدة المحاسبية في نظام KWPOS. يغطي دليل الحسابات، القيود اليومية، المصروفات (رواتب + إدارية)، والتقارير المالية الرئيسية: ميزان المراجعة، قائمة الدخل، الميزانية العمومية، التدفقات النقدية، الأستاذ العام، تقرير ضريبة القيمة المضافة، وكشف حساب العميل. مكتوب لغير المختصين تقنياً مع الإشارة إلى الملفات المصدرية.

الملف: `src/components/accounting/chart-of-accounts-tab.tsx`. الـ API: `src/app/api/accounts/route.ts`.

| النوع (Code) | الاسم العربي | لون | اتجاه الرصيد الطبيعي |

### الحسابات الجذرية (Root Accounts)

الحسابات `1000`, `2000`, `3000`, `4000`, `5000` هي **حسابات جذرية** (لا أصل لها). تُعرَض كرؤوس للشجرة. رصيدها = مجموع أرصدة جميع أبنائها (محسوب خادمياً في `accounts/route.ts` عبر `sumSubtree`).

### إضافة / تعديل / حذف حساب

| العملية | الطريقة | القيود |
|---|---|---|
| **إضافة حساب رئيسي** | زر «حساب رئيسي» أعلى الشجرة | يطلب اختيار النوع + الرمز + الاسم |
| **إضافة حساب فرعي** | زر «+» بجانب أي حساب | النوع يُورَّث تلقائياً من الأب |
| **تعديل** | API `PUT /api/accounts/[id]` | يسمح بتغيير الاسم/الرمز/الرصيد |
| **حذف** | API `DELETE /api/accounts/[id]` | ممنوع للحساب النظامي (`isSystem`)، أو الذي له أبناء، أو مرتبط بمصروف |

> **الحسابات النظامية** (`isSystem = true`) لا يمكن حذفها. الحسابات الافتراضية المُنشأة من `seed/route.ts` جميعها نظامية.

### التحقق من تفرّد الرمز

عند الإضافة، يفحص الخادم إن كان الرمز مستخدماً مسبقاً. إذا نعم، يُرجَع `code-exists` (HTTP 409) ويُعرَض للمستخدم: «رمز الحساب مستخدم مسبقاً».

## 3. القيود اليومية (Journal Entries)

الملف: `src/components/accounting/journal-tab.tsx`. الـ API: `src/app/api/journal-entries/route.ts`. مكتبة القيد: `src/lib/journal.ts`.

### مصادر القيود (`sourceType`)

| القيمة | الوصف | لون الشارة |
|---|---|---|
| `SALE` | قيد مبيعات (تلقائي عند إتمام فاتورة) | أخضر |
| `EXPENSE` | قيد مصروف (تلقائي عند تسجيل راتب/مصروف) | كهرماني |
| `PURCHASE` | قيد مشتريات/سداد مورد | أزرق سماوي |
| `MANUAL` | قيد يدوي من المحاسب | رمادي |

### شكل القيد الواحد (`JournalEntry`)

| الحقل | الوصف |
|---|---|
| `entryNo` | رقم القيد، بصيغة `JE-YYYYMMDDHHmmss-XXXX` (مولَّد في `nextEntryNo`) |
| `date` | تاريخ القيد |
| `sourceType` | أحد القيم أعلاه |
| `sourceId` | معرّف المستند الأصلي (Sale/Expense/PO) إن وُجد |
| `description` | وصف حر |
| `totalDebit` / `totalCredit` | مجموع المدين والدائن (يجب أن يتساوييا) |
| `lines` | أسطر القيد (`accountCode`, `debit`, `credit`, `description`) |

### القيد اليدوي (Manual Journal)

الملف: `src/components/accounting/manual-journal-dialog.tsx`. الـ API: `src/app/api/journal-entries/manual/route.ts`.

| الحقل | الإلزام | الملاحظات |
|---|---|---|
| **الوصف** | نعم | وصف القيد |
| **التاريخ** | نعم | افتراضياً اليوم |
| **الأسطر** | 2 على الأقل | كل سطر: حساب + مدين + دائن + وصف اختياري |
| **توازن المدين/الدائن** | إلزامي | يُعطَّل زر الحفظ إذا الفرق > 0.001 |

> **فحص التوازن:** في `src/lib/journal.ts` (دالة `createJournalEntry`) يُطلَق خطأ `journal-not-balanced` إذا `|totalDebit − totalCredit| > 0.001`. القيد لا يُحفَظ.

### قائمة القيود

- تعرض آخر **100 قيد** (`take: 100`) مرتبة من الأحدث إلى الأقدم.
- بحث مباشر في `entryNo` و`description`.
- ترقيم صفحات 10 قيود لكل صفحة.
- زر **Excel** يصدّر القيود عبر `/api/excel/export?type=journal`.
- الضغط على أي قيد يفتح **نافذة التفاصيل** (`journal-entry-modal.tsx`) تعرض كل أسطر القيد.

### مزامنة الأرصدة بعد القيد

عند إنشاء قيد **خارج معاملة** (مثل القيد اليدوي)، تُحدَّث أرصدة الحسابات فوراً داخل `createJournalEntry` (السطر `shouldSyncBalance`). أما عند إنشائه **داخل معاملة** (مثل قيد البيع أو المصروف)، يُتخطَّى تحديث الرصيد داخل المعاملة لتفادي إطالة قفل الاتصال، ويُترك للمعاملة أن تستدعي `syncAccountBalances` بعد الالتزام.

## 4. المصروفات (Expenses)

الملف: `src/components/accounting/expenses-tab.tsx`. الـ API: `src/app/api/expenses/route.ts` و`src/app/api/expenses/[id]/route.ts`.

### نوعا المصروف

| النوع | الحقول المطلوبة | الحساب المدين |
|---|---|---|
| **`SALARY`** (راتب) | `employeeName`, `amount`, `payDate` | 5010 (الرواتب) |
| **`ADMIN`** (مصروف إداري) | `title`, `category`, `amount`, `date` | 5020/5030/5040/5050/5090 حسب الفئة |

### فئات المصروف الإداري

| الفئة | الحساب |
|---|---|
| الإيجار | 5020 |
| المرافق | 5030 |
| الاشتراكات | 5040 |
| التسويق | 5050 |
| أخرى | 5090 |

### حقول مشتركة

- **حساب الدفع** (`paymentAccountId`): يجب أن يكون حساب أصول (ASSET). افتراضياً يقترح أول حساب أصول غير الجذر `1000`. في الواجهة يُميَّز `1010` (النقدية) بأيقونة نقد، و`1020` (البنك) بأيقونة بطاقة.
- **ملاحظة** (اختياري): نص حر يُحفَظ مع المصروف.

### القيد المحاسبي الناتج

عند تسجيل مصروف، يُنشِئ الخادم قيداً تلقائياً (في `expenses/route.ts`):

```
مدين: حساب المصروف (5010 أو 5020..5090)  — قيمة المصروف
دائن: حساب الدفع (1010 أو 1020)          — قيمة المصروف
```

الوصف التلقائي: `صرف راتب — {اسم الموظف}` أو `مصروف إداري — {العنوان} ({الفئة})`.

### الحذف (عكس القيد)

عند حذف مصروف (`DELETE /api/expenses/[id]`):

1. يُبحَث عن قيد `EXPENSE` مرتبط بـ`sourceId = id`.
2. يُعكَس تأثير كل سطر على رصيد الحساب (`decrement: debit − credit`).
3. يُحذَف القيد، ثم يُحذَف سجل المصروف.
4. تتم العملية داخل `db.$transaction` لضمان الذرية.

### قائمة المصروفات

- آخر **200 مصروف** (`take: 200`).
- لكل بند: الأيقونة (راتب = بنفسجي، إداري = كهرماني)، الاسم/العنوان، الفئة، حساب المصروف، حساب الدفع، التاريخ، المبلغ، زر حذف.
- حذف المتفائل (Optimistic): يُحذَف المصروف من الواجهة فوراً قبل ردّ الخادم.

## 5. التقارير المالية

### 5.1 ميزان المراجعة (Trial Balance)

الملف: `journal-tab.tsx` → `TrialBalanceTab`. الـ API: `src/app/api/trial-balance/route.ts`.

| العمود | الوصف |
|---|---|
| الرمز | رمز الحساب |
| اسم الحساب | الاسم |
| النوع | ASSET / LIABILITY / EQUITY / REVENUE / EXPENSE |
| مدين | الرصيد الموجب (للأصول/المصروفات) |
| دائن | الرصيد السالب بالقيمة المطلقة (للخصوم/الحقوق/الإيرادات) |

في تذييل الجدول يظهر **مجموع المدين** و**مجموع الدائن**. شارة أعلى الجدول:
- ✓ **متوازن** (أخضر) إذا `|totalDebit − totalCredit| < 0.001`.
- ✗ **غير متوازن** (أحمر) إن اختلفت.

> **ملاحظة:** الحسابات ذات الرصيد 0 لا تظهر في ميزان المراجعة.

### 5.2 قائمة الدخل (P&L)

الملف: `pnl-tab.tsx`. الـ API: `src/app/api/financial-reports/route.ts`.

تتألف من بطاقات:

| البطاقة | الحساب/الحساب |
|---|---|
| **إجمالي الإيرادات** | جميع المبيعات (POS + Shopify) في الفترة |
| **(−) تكلفة البضاعة المباعة (COGS)** | مجموع (كمية × costPrice) لكل بند مبيعات |
| **(=) مجمل الربح** | الإيرادات − التكلفة |
| **(−) الرواتب** | مصروفات النوع SALARY |
| **(−) المصروفات الإدارية** | مصروفات النوع ADMIN |
| **(=) صافي الربح** | مجمل الربح − إجمالي المصروفات |

بالإضافة إلى **بطاقة رئيسية** كبيرة تعرض صافي الربح بالقيمة المطلقة، وعدد الفواتير في الفترة، و**تفصيل المصروفات** (شريط أفقي لكل فئة مع نسبتها المئوية من إجمالي المصروفات).

الفلاتر: `from` و`to` تاريخيان، مع زر «كل الفترات».

### 5.3 الميزانية العمومية (Balance Sheet)

الملف: `balance-sheet-tab.tsx`. الـ API: `src/app/api/financial-reports/balance-sheet/route.ts`.

ثلاثة أعمدة:

| العمود | الحسابات | الإجمالي |
|---|---|---|
| **الأصول** | كل ASSET برصيده الموجب | `assetsTotal` |
| **الخصوم** | كل LIABILITY بالقيمة المطلقة | `liabTotal` |
| **حقوق الملكية** | كل EQUITY بالقيمة المطلقة | `eqTotal` |

شارة في الأعلى:
- ✓ **الميزانية متوازنة** (أخضر) إذا `|assets − (liabilities + equity)| < 0.01`.
- ✗ **غير متوازنة** (أحمر) إن اختلفت.

زر تصدير Excel/PDF عبر `ExportToolbar`.

### 5.4 التدفقات النقدية (Cash Flow)

الملف: `cash-flow-tab.tsx`. الـ API: `src/app/api/financial-reports/cash-flow/route.ts`.

تُحسَب التدفقات من **أسطر القيود على حسابي النقدية والبنك فقط** (1010, 1020):
- **تدفق داخل** (+): سطر مدين على حساب النقدية/البنك.
- **تدفق خارج** (−): سطر دائن على حساب النقدية/البنك.
- يُجمَّع حسب `sourceType` (مبيعات/مصروفات/مشتريات/أخرى).

| البطاقة | الوصف |
|---|---|
| **رصيد افتتاحي** | مجموع (مدين − دائن) لكل أسطر النقدية/البنك **قبل** `from` |
| **صافي التدفق** | totalIn − totalOut للفترة |
| **رصيد ختامي** | رصيد افتتاحي + صافي التدفق |

الفلاتر: `from`, `to`. تصدير Excel/PDF متاح.

### 5.5 الأستاذ العام (General Ledger)

الملف: `general-ledger-tab.tsx`. الـ API: `src/app/api/financial-reports/general-ledger/route.ts`.

يعرض **حركة حساب واحد** خلال فترة، مع **رصيد جارٍ** لكل سطر.

| الحقل | الوصف |
|---|---|
| الحساب | Combobox لاختيار أي حساب (إلزامي) |
| من / إلى | نطاق تاريخي اختياري |

كل سطر في الجدول: التاريخ، رقم القيد، الوصف، مدين، دائن، **الرصيد بعد السطر** (running balance).

بطاقات علوية:
- **الرصيد الافتتاحي** (أصفر): مجموع (مدين − دائن) قبل `from`.
- **الرصيد الجاري** (أخضر): الرصيد بعد آخر سطر في الفترة.
- **معلومات الحساب**: الرمز + الاسم + النوع.

> يجب اختيار حساب لعرض البيانات (لا توجد بيانات افتراضية).

### 5.6 تقرير ضريبة القيمة المضافة (VAT)

الملف: `vat-report-tab.tsx`. الـ API: `src/app/api/financial-reports/vat/route.ts`.

| البطاقة | المصدر | الوصف |
|---|---|---|
| **ضريبة المخرجات** | `sale.taxAmount` (مجموع) | الضريبة المحصَّلة من المبيعات |
| **ضريبة المدخلات** | فواتير المشتريات POSTED + أوامر استلام بلا فاتورة مرتبطة | الضريبة المدفوعة على المشتريات |
| **صافي الضريبة** | output − input | موجب = مستحقة السداد، سالب = قابلة للاسترداد |
| **إجمالي المبيعات/المشتريات** | subtotal / total | قيمة المبيعات والمشتريات بدون الضريبة |

شارة في بطاقة الصافي:
- **مستحقة السداد** (أحمر) إذا `netVat ≥ 0`.
- **قابلة للاسترداد** (أخضر) إذا `netVat < 0`.

الفلاتر: `from`, `to`. تصدير Excel/PDF.

### 5.7 كشف حساب العميل (Customer Statement)

الملف: `customer-statement-tab.tsx`. الـ API: `src/app/api/customers/[id]/statement/route.ts`.

| الحقل | الوصف |
|---|---|
| العميل | Combobox إلزامي |
| من / إلى | نطاق تاريخي اختياري |

لكل حركة: التاريخ، النوع (فاتورة/مرتجع)، المرجع (`invoiceNo` أو `refundNo`)، مدين (للفاتورة)، دائن (للمرتجع)، **الرصيد بعد الحركة**.

بطاقات علوية:
- **الرصيد الافتتاحي**: رصيد العميل قبل `from`.
- **الرصيد الختامي**: الرصيد بعد آخر حركة في الفترة.
- **بيانات العميل**: الاسم + الهاتف.

تصدير Excel/PDF متاح عبر `ExportToolbar`.

## 6. ملاحظات للمحاسب

- **القيد المتوازن إلزامي:** لا يمكن حفظ قيد غير متوازن (يدوياً أو تلقائياً).
- **الفرق بين ميزان المراجعة والميزانية:** ميزان المراجعة يفحص التوازن (مدين = دائن). الميزانية تفحص المعادلة المحاسبية (أصول = خصوم + حقوق ملكية). الاثنان مفروض أن يكونا متوازنين.
- **عكس المصروف:** حذف مصروف يعكس قيده تماماً (دائن بدل مدين) ويعيد الأرصدة لحالتها السابقة. لا يمكن حذف مصروف مرتبط بقيد SALE.
- **تأثير الفلتر الزمني:** قائمة الدخل تستخدم `createdAt` للمبيعات و`payDate`/`date` للمصروفات حسب النوع. لو دفعت راتباً بتاريخ قديم، سيظهر في قائمة دخل تلك الفترة لا فترة اليوم.
- **عدم ظهور حساب جذر في القيود:** حوار القيد اليدوي يستثني رموز `1000`, `2000`, `3000`, `4000`, `5000` — لا يمكن التقييد مباشرة على حساب جذر.
